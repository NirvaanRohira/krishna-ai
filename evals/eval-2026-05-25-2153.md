This is a question about the nature of *karma* and *prakriti*—the binding force of conditioned nature. The life domain is the bondage of action rooted in desire and aversion, which the Gita calls *icchā-dveṣa*.

Let me tell you what the texts actually say. ### Scorer bugs (false negatives inflating perceived problems)

Three scoring dimensions produce incorrect results independent of model quality.

**1. `followUpQuestion` — markdown bold/italic strips the trailing `?`**

The scorer splits on `[.!?]+` and checks if the last fragment contains `?`. When the model wraps the follow-up in `**...**` or `*...*` markdown, the split leaves `**` as the trailing fragment, hiding the `?`.

Affected questions: **Q1, Q4, Q5, Q8** — all four had a follow-up question that the scorer missed because of bold or italic wrappers. Fix: strip markdown before the check, or use a regex that finds any `?` within the last 300 characters of the response.

```
const hasFollowUp = /\?.{0,20}$/.test(text.trim())
```

**2. `textualCitation` — ordinal words not matched by `/skandha \d/`**

The regex requires a digit after "skandha", "chapter", or "verse". The model consistently writes word-form ordinals: "third skandha", "tenth skandha", "eleventh skandha", "fifth skandha". None of these match.

Affected: **Q1, Q3, Q7, Q8** — all cited specific textual locations using word-form numbers but scored only 1/2.

Fix: add an ordinal word pattern:
```
/chapter (?:\d+|[a-z]+)|skandha (?:\d+|[a-z]+)|verse (?:\d+|[a-z]+)|\d+\.\d+/
```

**3. `expectedSignals` — literal substring matching breaks on reformatting**

Q4's signal `'chapter 2'` never matches because the model writes "the second chapter, verse 47" — accurate citation, different word order. Fix: expand signals with synonyms or use a looser match (e.g., "second chapter" counts for `chapter 2`).
